What is Sass?

Sass (Syntactically Awesome Stylesheets) extends CSS with variables, nesting, mixins, functions and more. You write .scss files and compile them to plain CSS for the browser.

Why use it?

  • Variables for colors and spacing
  • Nesting to keep related rules together
  • Mixins to reuse blocks of styles
  • Partials & imports for modular styles

SCSS Example

Open CSS/day7.scss to see the source. Here is a shortened snippet:

$accent: #5b8def;

@mixin button($bg){
  background: $bg;
  color: #fff;
  padding: 8px 12px;
  border-radius: 6px;
}

.card{
  border-radius:8px;
  padding:16px;
  h3{margin-top:0}
  .btn{ @include button($accent); }
}

That SCSS compiles to regular CSS (see the provided CSS/day7.css), which the browser understands.

How to compile

Using the official Dart Sass on your machine:

sass CSS/day7.scss CSS/day7.css
# or for watch mode
sass --watch CSS/day7.scss:CSS/day7.css

If you don't want to install anything now, edit the SCSS for learning and use the provided compiled day7.css to preview changes.
