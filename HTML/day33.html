1. What Is An Error?

An error happens when JavaScript doesn't understand your code or when something goes wrong while it runs. Think of it like giving a robot instructions it can't follow.

  • SyntaxError: You typed something wrong (like missing a parenthesis).
  • ReferenceError: You used a variable that doesn't exist yet.
  • TypeError: You tried to do something with the wrong kind of value.
// ReferenceError example
console.log(score); // score was never defined

2. Catching Errors with try ... catch

The try block is where you put code that might break. If it breaks, the catch block runs instead of crashing everything.

try {
  JSON.parse('{ broken json');
  console.log('This line will NOT run');
} catch (error) {
  console.log('Oops! There was a problem.');
  console.log(error.name);   // Error type
  console.log(error.message); // What happened
}

3. Creating Your Own Errors

You can throw an error on purpose when something is wrong (like a missing username). This helps stop bad data.

function checkAge(age) {
  if (age < 0) {
	throw new Error('Age cannot be negative');
  }
  return 'Age is fine!';
}

try {
  checkAge(-5);
} catch (err) {
  console.log(err.message); // Age cannot be negative
}

4. Debugging: Finding the Bug

Debugging means hunting the problem. Here are simple steps:

  1. Read the error message (it tells you the file + line number).
  2. Go to that line in your code.
  3. Add console.log() to inspect values.
  4. Fix the mistake. Test again.
  5. Remove extra logs when done.

Open your browser DevTools (usually F12 or Ctrl + Shift + I) then click the Console tab to see messages.

// Debugging with console.log
function add(a, b) {
  console.log('a =', a, 'b =', b); // helps you see values
  return a + b;
}
add(2, 5);
