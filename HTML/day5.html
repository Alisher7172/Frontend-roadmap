Day 5 — How CSS Styling Actually Works

The cascade & sources

Styles come from multiple sources: browser defaults (user-agent), user styles, and author styles (your CSS). The cascade decides which rules apply when there are conflicts — later rules can override earlier ones depending on specificity and importance.

Specificity (short version)

When multiple rules target the same element the one with higher specificity wins. Specificity roughly ranks like this (higher wins):

  1. Inline styles (style="...")
  2. ID selectors (#id)
  3. Class, attribute and pseudo-class selectors (.class, [attr], :hover)
  4. Element and pseudo-element selectors (div, ::before)

Note: !important raises priority above normal rules — use sparingly.

Inheritance

Some properties (like color, font-family) inherit from parent elements; others (like padding, margin) do not.

Computed style

The browser computes a final style for each element after applying cascade, specificity, inheritance, and layout. Use DevTools > Elements > Styles to inspect the computed values.
