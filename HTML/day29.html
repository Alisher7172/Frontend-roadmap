ES6+ Features Explained

let & const

let and const are block-scoped. let allows reassignment, const does not.
var is function-scoped and can cause bugs due to hoisting.
Prefer const for values that never change, and let for those that do.
Example: const PI = 3.14; (cannot be reassigned)

Arrow Functions

Arrow functions (=>) provide a shorter syntax and inherit this from their parent scope.
They do not have their own this, arguments, or super.
Useful for callbacks and array methods.
Example: const add = (a, b) => a + b;

Spread & Rest Operators

Spread (...) expands arrays/objects.
Rest (...) collects multiple arguments into an array.
Example: const arr2 = [...arr1, 4, 5]; (spread)
Example: function sum(...nums) {} (rest)

Destructuring

Destructuring lets you extract values from arrays or properties from objects into distinct variables.
Example: const [a, b] = [1, 2];
Example: const {name, age} = user;

Classes

ES6 classes provide a clear syntax for creating objects and inheritance.
Use constructor for initialization, and methods for behavior.
Example: class Dog extends Animal { speak() { ... } }

Template Literals

Template literals use backticks (`) for multi-line strings and variable interpolation.
Example: const msg = `Hello, ${name}!`;

Default Parameters

Functions can have default parameter values.
Example: function greet(name = 'Guest') { ... }

Promises

Promises handle asynchronous operations and avoid callback hell.
Example: fetch(url).then(res => res.json()).catch(err => ...);

Modules (import/export)

ES6 modules allow you to split code into reusable files.
Example: import { add } from './math.js';
Example: export function add(a, b) { return a + b; }
