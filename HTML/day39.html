Day 39 — Hosting on Netlify

A short, practical guide to hosting static sites (and modern frontends) on Netlify.

What is Netlify?

Netlify is a platform that makes it easy to deploy static sites and JAMstack apps. It provides continuous deployment from Git, one-click drag & drop deploys, custom domains, HTTPS, environment variables, redirects, and deploy previews out of the box.

Why use Netlify?

  • Fast CDN-backed hosting with automatic HTTPS.
  • Easy Git integration (GitHub, GitLab, Bitbucket).
  • Deploy previews for PRs and instant rollbacks.
  • Simple configuration with netlify.toml or a _redirects file.

Quick start (3 ways)

1) Drag & Drop (fastest)

Build your static site (folder with an index.html) and drop the folder into the Netlify dashboard's "Sites" → "Add new site" → "Deploy manually" area.

2) Connect your Git repository (recommended)

  1. Push your site to GitHub/GitLab/Bitbucket.
  2. In Netlify dashboard, choose "New site from Git" and authorize the provider.
  3. Select repo, set build command (e.g., npm run build) and publish directory (e.g., dist or build).
  4. Netlify will build and deploy on every push.

3) CLI / API

You can also use the Netlify CLI to deploy from your machine or CI: npm i -g netlify-cli then netlify deploy.

Common configs

netlify.toml (example)

[build]
  publish = "dist"
  command = "npm run build"

[[redirects]]
  from = "/api/*"
  to = "https://example.com/api/:splat"
  status = 200

_redirects (simple redirects example)

/old-path  /new-path  301
/blog/*    /index.html  200

Environment variables

Set them in the Netlify UI or via the CLI for API keys, build-time flags, and secrets.

Tips & common issues

  • If your site uses a build step, ensure the build command and publish directory are correct.
  • Use netlify.toml to keep site-specific settings in the repo.
  • For single-page apps (React/Vue), configure redirects to serve index.html for client-side routes.
  • Check deploy logs in the Netlify dashboard when builds fail — they show the exact command output.

Example: deploy from GitHub (commands)

git init
git add .
git commit -m "Initial site"
git remote add origin git@github.com:youruser/yourrepo.git
git push -u origin main
