Understanding CSS Units

A quick guide to `px`, `%`, `rem`, `em`, `vw`, `vh`, and `ch` for responsive web design.

1. Pixels (`px`)

Pixels are absolute units. They define a fixed size based on the physical pixels of the display. While straightforward, they can lead to accessibility issues as they don't scale with user preferences (like browser font size settings) and can make layouts less flexible on different screen sizes.

16px text size

24px text size

font-size: 16px; and font-size: 24px;

2. Percentages (`%`)

Percentages are relative units. They are relative to the parent element's property. For font sizes, it's relative to the parent's font size. For widths, it's relative to the parent's width. This makes them excellent for creating fluid and responsive layouts.

This div is 50% of its parent's width.

100% text size (relative to parent)

150% text size (relative to parent)

width: 50%;, font-size: 100%;, font-size: 150%;

3. Root EM (`rem`)

`rem` stands for "root em". It's a relative unit that is always relative to the font size of the root HTML element (``). This makes it incredibly powerful for responsive typography and consistent scaling across your entire website, as changing the root font size will proportionally adjust all `rem` values.

Note: The default browser root font size is usually 16px. So, 1rem = 16px by default.

1rem text size

1.5rem text size

font-size: 1rem; and font-size: 1.5rem;

4. EM (`em`)

Similar to `rem`, `em` is a relative unit. However, `em` is relative to the font size of its parent element. This can sometimes lead to compounding issues (where sizes multiply unexpectedly) if not managed carefully, but it's useful for scaling elements within a specific component.

Parent font size is 20px

1em text (20px)

1.2em text (24px)

Nested div, font size is 0.8em (16px)

1.5em text (24px, relative to its new parent)

font-size: 1em; and font-size: 1.2em;

5. Viewport Units (`vw`, `vh`)

Viewport units are relative units that are based on the size of the viewport (the browser window).

  • `vw` (viewport width): 1vw is 1% of the viewport's width.
  • `vh` (viewport height): 1vh is 1% of the viewport's height.

These are incredibly useful for creating elements that scale perfectly with the browser window, ensuring a truly responsive design.

This box is 50vw wide and 10vh tall.

This text is 3vw (3% of viewport width)

width: 50vw;, height: 10vh;, font-size: 3vw;

6. Character Unit (`ch`)

The `ch` unit is a relative unit that represents the width of the "0" (zero) character in the current font. It's particularly useful for setting widths of text containers to ensure a consistent number of characters per line, which can improve readability, especially for monospaced fonts.

This paragraph has a width of 40ch, meaning it tries to fit 40 '0' characters. This is great for code blocks or fixed-width text.

width: 40ch;
